# 命名规则
DynamoDB 中的表、属性和其他对象必须具有名称
命名规则：
+ 所有名称都必须使用 UTF-8 进行编码，并且区分大小写。

+ 表名称和索引名称的长度必须介于 3 到 255 个字符之间，而且只能包含以下字符：

   +  a-z

    + A-Z

    + 0-9

    + _ (下划线)

    + \- (短划线)

    + . (圆点)

+ 属性名称的长度必须至少为 1 个字符，但不得超过 64 KB。

以下属性名称的长度不得超过 255 个字符：

+ 辅助索引分区键名称。

+ 辅助索引排序键名称。

+ 任意用户指定的投影属性的名称 (仅适用于本地二级索引)。
